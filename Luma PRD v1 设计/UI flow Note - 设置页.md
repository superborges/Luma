# UI flow Note - 设置页

## 1. 流程名称

- 名称：设置
- 所属模块：辅助页（首页菜单进入）
- 使用频率：低

## 2. 这个流程要解决什么

- 用户来到这个流程时的目标：配置 AI 模型 / 评分策略 / 导出默认值等长期生效的偏好
- 用户最关心的信息：哪个 AI 模型在用、花了多少钱、默认导出去哪
- 用户最害怕发生的问题：
    - API key 泄露
    - 没设预算，AI 评分把钱花光
    - 配置错了模型，所有 Session 评分都白做

## 3. 入口与出口

- 从哪里进入：首页右上角设置图标 / 系统菜单"Luma > 偏好设置"（⌘,）
- 完成后去哪里：返回原页面
- 中途取消后回到哪里：原页面（未点保存的更改丢弃）

## 4. 页面步骤

### Step 1

- 页面名：设置页 - AI 模型
- 用户要做什么：管理 AI 视觉模型配置
- 系统展示什么：
    - 已配置模型列表（Claude/GPT/Gemini/本地等）
    - 每个模型显示：名称、协议、模型 ID、是否启用、角色（Primary/Premium）
    - 添加新模型按钮
    - 测试连接按钮
- 主按钮是什么：
    - "添加模型"
    - 单条模型的"启用 / 停用 / 测试 / 删除"

### Step 2

- 页面名：设置页 - 评分策略
- 用户要做什么：选择 AI 评分策略和预算上限
- 系统展示什么：
    - 三种策略：
        - 经济：仅用 Primary 模型按组打分
        - 平衡：Primary 打分 + Premium 对 Top 20% 详细分析
        - 最高质量：Premium 对全部高分照片详细分析
    - 预算上限输入框（默认 $5）
    - 当前 Session 已花费统计
- 主按钮是什么：单选策略

### Step 3

- 页面名：设置页 - 导出默认值
- 用户要做什么：设置导出默认目标与格式，避免每次重复配
- 系统展示什么：
    - 默认导出目录
    - 默认 Lightroom 自动导入目录
    - 默认未选中照片处理方式
- 主按钮是什么：各项的"浏览"按钮

### Step 4

- 页面名：设置页 - 关于（可选）
- 用户要做什么：查看版本/查看日志/打开诊断
- 系统展示什么：版本号、日志路径、性能诊断入口
- 主按钮是什么："打开性能诊断"（开发者用）

## 5. 数据规则

- 所有配置存 UserDefaults，API Key 存 macOS Keychain
- 模型配置可多个并存，但同一时刻 Primary/Premium 各只有一个生效
- 设置变更立即生效，不需要重启
