# Build Spec

## 1. 版本信息

- 产品：Luma
- 版本：1.0
- 日期：2026.04

## 2. 本次版本目标

- 本次版本要解决的核心问题：导入和选片流程/导出流程/兼容 Iphone生态
- 本次版本最重要的用户任务：选片和导出
- 本次版本完成后，用户能做到什么：
    - 可以导入 SD卡和 iPhone 照片
    - AI 智能分组，可以连拍组选片
    - AI 提供智能修图建议
    - 可以导出到普通目录和 iPhone 相册
    - 可以支持导出视频

## 3. 功能范围

### 本次包含

- SD 卡和 iPhone 相册导入
- AI 智能分组
- 智能识别连拍组
- AI 智能图片打分
- AI 提供修图建议
- 未选中照片支持归档，且压缩成视频作为纪念
- 可以导出到 iPhone 相册

### 本次不包含

- AI 智能修复
- 云备份等高级功能

## 4. 用户主路径

1. 用户进入：Import Session 列表界面
2. 用户看到：历史的 Import Session，可以新建 Import Session from 本地目录/sd 卡/iPhone 相册
3. 用户执行：
    1. 用户新建 Import Session，从目录/sd 卡/iPhone 里导入照片，构建照片缓存，智能分组，进行选片，然后导入选中照片，归档未选中照片，未选中照片可以导出成视频留作纪念。
    2. 针对选中的照片，导出可以适配 iPhone 相册格式
    3. 调用 LLM 模型（claude/gpt5-4 等）给选中照片提供修图建议（直接修图 v2 做）

## 5. 页面清单

- 首页：Import Session 的列表页
- 新建页：新建 Import Session 的 hub 页，支持 普通目录/sd卡目录/iPhone 相册
- 选片页：一个Session 里的照片进行智能分组，在组内进行照片选择，自动聚合连拍组。建议是三栏布局，左边是分组，中间是照片和操作区，支持单张和网格，右边是照片信息区。
- 导出页：新建导出 session，针对未选中的照片可以配置压缩，且可以将它们压缩成一个视频归档。

## 6. 每页核心任务

### 首页

- 页面目标：展示所有 Import Session
- 主操作：查看所有 Session 列表，进入一个 Session 进行 culling 选片操作，可以 close 一个 session
- 次操作：新建一个 Import Session

### 新建页

- 页面目标：新建 Import Session
- 主操作：新建一个 Import Session，支持 普通目录/sd 卡/iPhone
- 次操作：无

### 选片页

- 页面目标：特定 Import Session的 culling 选片
- 主操作：切换不同分组，在分组里，依次对照片进行 选中/待定/拒绝 操作
- 次操作：可以补导入一些照片

### 导出页

- 页面目标：导出 Session 里的照片
- 主操作： 选中的照片可以导出到普通目录/iPhone 相册
- 次操作：未选中照片可以压缩，然后转成视频归档。

## 9. 数据与对象

- 核心对象：Import Session/分组/连拍组/导出任务
- 对象间关系：Import Session 是核心对象（去掉 expedition 概念），它包括多个分组，分组里有n 个连拍组和m 个普通照片，1 个 Session 可以有多个导出任务。
- 用户会修改哪些内容：新建Import Session，选片，导出，归档等
