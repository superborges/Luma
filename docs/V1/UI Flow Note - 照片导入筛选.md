# UI Flow Note — 照片导入筛选增强（F3）

## 1. 流程名称

- 名称：Mac·照片 App 导入筛选
- 所属模块：导入（`AppKitPhotosImportPicker`）
- 使用频率：高（iPhone/iCloud 用户每次导入都用）

## 2. 这个流程要解决什么

- 用户来到这个流程时的目标：从 Mac 照片图库中精准筛选一批照片导入 Luma，而不是「瞎抓最近 N 张」
- 用户最关心的信息：
  - 能不能只导最近一次旅行的？（时间范围）
  - 能不能只导某个相册？（用户相册）
  - 大概多少张、占多少空间？（预估）
  - 之前导过的会不会重复？（去重）
- 用户最害怕发生的问题：
  - 选了「全部」，5 万张照片卡死
  - 导入了已导过的照片，Session 里出现重复
  - 不知道 iCloud 未下载的照片会怎样

## 3. 入口与出口

- 从哪里进入：首页 → 新建 → 选「Mac·照片 App」→ 授权通过后弹出筛选面板
- 完成后去哪里：导入进度 → 选片工作区
- 中途取消后回到哪里：首页

## 4. 页面步骤

### Step 1 — 筛选面板

- 页面名：照片导入筛选（`NSAlert` 多段控件）
- 用户要做什么：配置筛选条件
- 系统展示什么：
  - **时间范围**：预设（最近 7 天 / 30 天 / 90 天 / 不限）+ 自定义日期区间
  - **相册**（可选，二选一）：
    - 智能相册：最近添加 / 收藏 / 截图 / 自拍 / 实况照片 / 连拍
    - 用户自建相册（下拉列表，按最近修改排序）
  - **媒体类型**：全部（含 Live）/ 仅静态 / 仅 Live Photo
  - **上限**：默认 500，可调
  - **去重开关**：跳过当前 Project 已导入过的（默认开）
  - **预估**（实时刷新）：匹配 X 张，预估占用 Y MB（其中云端 Z 张）
- 主按钮：「导入」

### Step 2 — 确认与导入

- 页面名：确认弹窗（若预估为 0 或全部被去重跳过）
- 用户要做什么：确认或返回调整
- 系统展示什么：
  - 若匹配 0 张：「当前条件下没有可导入的照片，请调整筛选条件」
  - 若全部已导入：「这些照片已经在当前 Project 中了」
- 主按钮：「返回调整」

### Step 3 — 导入进度

- 页面名：导入中（与其他来源一致）
- 用户要做什么：等待
- 系统展示什么：来源名称（含筛选摘要）、进度条
- 主按钮：完成后进入选片

## 5. 页面布局原则

- 主区域放什么：筛选控件纵向排列，自上而下为时间 → 相册 → 类型 → 上限 → 去重
- 侧边区域放什么：无（单页弹窗）
- 次要信息放哪里：预估信息在底部，实时刷新
- 操作区固定还是浮动：固定在弹窗底部（取消 / 导入）

## 6. 交互规则

- 什么时候允许快速操作：选择预设时间范围是即时切换
- 什么时候必须确认：匹配为 0 时拦截
- 什么时候给提示：云端照片数量 > 0 时提示「X 张仅在 iCloud，导入阶段使用本地缓存版」
- 什么时候自动切换下一项：不自动切换（用户主动点「导入」）
- 是否支持快捷键：Enter = 导入，Esc = 取消
- 是否支持撤销：否（取消导入即可）

## 7. 状态说明

### 默认状态

- 内容：「最近 30 天 / 无相册 / 全部类型 / 上限 500 / 去重开」
- 操作：调整任一字段 → 预估实时刷新

### 空状态

- 提示：匹配 0 张时，「导入」按钮置灰 + 底部提示
- 引导动作：「试试扩大时间范围或换一个相册」

### 加载状态

- 是否阻塞：预估计算不阻塞交互，spinner 在预估数字旁
- 骨架屏还是 loading：数字区域 spinner

### 错误状态

- 提示位置：弹窗内
- 是否允许重试：权限问题展示 `PhotosAccessGuidance`，可跳转系统设置
- 是否保留当前上下文：是

## 8. 设计注意点

- 视觉重点：预估数字要醒目（张数 + 磁盘占用）
- 不要过度设计的地方：相册选择用标准下拉即可，不需要缩略图预览
- 必须清晰的地方：「仅读本地缓存版，不会拉 iCloud」的文案要可见

## 9. 开发注意点

- 哪些可以先 mock：无（`PhotosImportPlanner` 已有 estimate / userAlbums 等）
- 哪些交互先做最简版：自定义日期区间可用两个 DatePicker，不需要日历组件
- 哪些动画先不要做：预估刷新动画
