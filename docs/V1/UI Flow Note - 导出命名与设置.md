# UI Flow Note — 导出文件命名 + 设置页增强（F4 + F5）

## 1. 流程名称

- 名称：导出文件命名规则 + 设置页增强
- 所属模块：导出页（`ExportPanelView`）+ 设置页（`SettingsView`）
- 使用频率：中

## 2. 这个流程要解决什么

- 用户来到这个流程时的目标：
  - 导出时控制输出文件的命名方式（保留原名、加日期前缀、自定义模板）
  - 调整全局偏好（分组灵敏度、默认导入目录、缓存策略）
- 用户最关心的信息：
  - 导出后的文件叫什么名？会不会和已有文件冲突？
  - 分组为什么把一个场景拆成两组/合成一组？能不能调？
- 用户最害怕发生的问题：
  - 导出文件全叫 `IMG_xxxx.jpg`，混在一起分不清
  - RAW+JPEG 配对命名被打散

---

## Part A — 导出文件命名规则（F4）

### 3. 入口与出口

- 从哪里进入：导出面板 → 配置目标 Section 内
- 完成后去哪里：导出执行
- 中途取消后回到哪里：选片页

### 4. 页面步骤

#### Step 1 — 选择命名规则

- 页面名：导出面板 - 文件命名
- 用户要做什么：从预设中选择或输入自定义模板
- 系统展示什么：
  - **Picker**（Segmented 或下拉）：
    - 保留原名（`IMG_1234.jpg`）← 默认
    - 日期前缀（`2026-01-15_IMG_1234.jpg`）
    - 自定义模板（`{date}_{group}_{original}`）
  - **预览**：选择后实时展示一个示例文件名
  - **说明**：RAW+JPEG 配对时，两者使用相同主文件名、不同扩展名
- 主按钮：无（内联在导出配置中）

#### 可用模板变量

| 变量 | 含义 | 示例 |
|------|------|------|
| `{original}` | 原始文件名（不含扩展名） | `IMG_1234` |
| `{date}` | 拍摄日期 `yyyy-MM-dd` | `2026-01-15` |
| `{datetime}` | 拍摄时间 `yyyy-MM-dd_HHmmss` | `2026-01-15_143022` |
| `{group}` | 所属分组名 | `清水寺` |
| `{seq}` | 组内序号（从 1 开始） | `003` |

### 5. 交互规则

- 选择后实时刷新预览（不需要保存按钮）
- 自定义模板输入框内有 placeholder：`{date}_{original}`
- 命名冲突时自动追加 `-2`、`-3`
- 命名规则随导出选项一起持久化到 UserDefaults

## Part B — 设置页增强（F5）

### 3. 入口与出口

- 从哪里进入：⌘, 或菜单 → Settings
- 完成后：关闭窗口

### 4. 新增设置项

#### 通用 Tab — 新增

| 设置项 | 控件 | 默认值 | 说明 |
|--------|------|--------|------|
| 分组时间阈值 | Picker（15 分钟 / 30 分钟 / 60 分钟） | 30 分钟 | 两张照片超过此间隔则分为不同组；下次导入生效 |
| 默认导入目录 | 路径 + 「选择…」按钮 | 无（每次手动选） | 新建导入时自动填充；仅对「普通目录」来源有效 |
| 缩略图缓存上限 | Stepper（100 / 200 / 500 / 1000） | 200 | `ThumbnailCache.countLimit`；超出后 LRU 淘汰 |

#### 导出默认值 Tab — 新增

| 设置项 | 控件 | 默认值 | 说明 |
|--------|------|--------|------|
| 默认文件命名规则 | Picker（保留原名 / 日期前缀 / 自定义） | 保留原名 | 新建导出时自动填充；可在导出面板覆盖 |

### 5. 交互规则

- 分组时间阈值修改后显示 inline 提示：「将在下次导入的分组中生效」
- 默认导入目录为空时，新建时仍弹 `NSOpenPanel`
- 缩略图缓存上限修改后立即生效（热更新 `countLimit`）
- 所有设置项持久化到 UserDefaults，键前缀 `Luma.`

### 6. 状态说明

#### 默认状态

- 内容：各字段显示当前值
- 操作：直接修改

#### 空状态

- 默认导入目录为空时显示「未设置」灰色文案

### 7. 设计注意点

- 视觉重点：分组阈值 Picker 文案要通俗（「15 分钟 — 密集拍摄」/「60 分钟 — 旅行跳点」）
- 不要过度设计的地方：缩略图缓存上限用 Stepper 即可，不需要滑块
- 必须清晰的地方：「下次导入生效」的 inline 提示

### 8. 开发注意点

- 哪些可以先 mock：无（直接读写 UserDefaults + 传参到 GroupingEngine / ThumbnailCache）
- 哪些交互先做最简版：自定义命名模板先支持固定变量集，不做正则校验
- 哪些动画先不要做：设置切换动画
