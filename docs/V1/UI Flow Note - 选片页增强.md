# UI Flow Note — 选片页增强（F1 + F2）

## 1. 流程名称

- 名称：选片右栏增强 + 快捷键补齐
- 所属模块：选片页（`CullingWorkspaceView`）
- 使用频率：高（每次选片全程使用）

## 2. 这个流程要解决什么

- 用户来到这个流程时的目标：快速判断每张照片好不好，高效做出 Pick/Reject/Pending 决策
- 用户最关心的信息：
  - 这张照片 AI 打了多少分？哪些维度好/差？
  - 是不是废片（模糊/过曝/欠曝）？
  - 连拍组里 AI 推荐哪张？
- 用户最害怕发生的问题：
  - 几百张照片一张张看完全靠肉眼，太慢
  - 连拍 5 张看不出区别，来回犹豫
  - 没有 U 键回退，误操作要去菜单找

## 3. 入口与出口

- 从哪里进入：Session 列表点击 Session → 选片工作区
- 完成后去哪里：导出页
- 中途取消后回到哪里：首页（选片状态自动保存）

## 4. 页面步骤

### Step 1 — 进入选片，右栏已展示评分

- 页面名：选片工作区
- 用户要做什么：浏览分组列表，选择一个分组开始
- 系统展示什么：
  - 左栏：智能分组列表（缩略图、组名、照片数、选片进度）
  - 中间区：当前分组照片网格
  - **右栏（新增）**：
    - EXIF 信息（继承 MVP）
    - **AI 评分卡**：总分（0-100）+ 五维子分条（清晰度/构图/曝光/色彩/主体）
    - **废片标签**：若有 issues，展示标签列表（如「模糊」「过曝」「抖动」）
- 主按钮：点击分组进入

### Step 2 — 单张决策（快捷键增强）

- 页面名：选片工作区 - 单张模式
- 用户要做什么：对照片做 Pick / Reject / Pending 操作
- 系统展示什么：
  - 中间区：大图预览
  - 右栏：AI 评分卡 + 废片标签 + EXIF
  - 底部状态栏：当前组进度（已选 X / 拒 Y / 待定 Z）
- **快捷键**：
  - `P` — 选中（Pick）
  - `X` — 拒绝（Reject）
  - `U` — 回待定（Pending）← **V1 新增**
  - `G` — 切换网格/单张模式 ← **V1 新增**
  - `←/→` — 上/下一张
  - `↑/↓` — 上/下一组
  - `Tab` — 下一组
  - `1-5` — 评分 1-5 星
  - `Space` — 待定（与 `U` 等效，保持 MVP 兼容）

### Step 3 — 连拍组内选优

- 页面名：连拍组展开视图
- 用户要做什么：在连拍组里挑出最好的一张
- 系统展示什么：
  - 组内所有照片并排展示
  - AI 推荐的最佳照片有**蓝色边框标记**
  - 每张照片下方显示 AI 总分
- 主按钮：**「采纳 AI 推荐」** ← **V1 新增**（一键将 AI 推荐设为 Picked，其余设为 Rejected）

### Step 4 — 完成选片

- 页面名：选片完成
- 用户要做什么：确认进度 → 去导出
- 系统展示什么：顶部进度（X 张已决策 / 共 Y 张），全部完成后「去导出」高亮
- 主按钮：「去导出」

## 5. 页面布局原则

- 主区域放什么：中间大图或网格，占最大宽度
- 侧边区域放什么：
  - 左栏：分组导航（固定宽度）
  - 右栏：EXIF → AI 评分卡 → 废片标签（纵向排列，评分卡可折叠）
- 次要信息放哪里：底部状态栏（组内进度）
- 操作区固定还是浮动：快捷键全局；右栏跟随选中照片刷新

## 6. 交互规则

- 什么时候允许快速操作：任何时候都可按 P/X/U 快速决策
- 什么时候必须确认：「采纳 AI 推荐」批量操作前无需二次确认（可撤销）
- 什么时候给提示：废片标签直接内联展示，不弹 toast
- 什么时候自动切换下一项：P/X/U 操作后自动跳下一张（到组末时跳下一组首张）
- 是否支持快捷键：是（全部列在 Step 2）
- 是否支持撤销：「采纳 AI 推荐」可通过再次手动 Pick/Reject 覆盖

## 7. 状态说明

### 默认状态

- 内容：右栏展示 EXIF + AI 评分 + 废片标签
- 操作：可切视图、快捷键决策

### 空状态

- 提示：照片无评分时显示「暂无评分」灰色文案
- 引导动作：无（评分在导入后自动运行）

### 加载状态

- 是否阻塞：否，照片优先展示，评分异步加载后刷新
- 骨架屏还是 loading：评分区域显示占位骨架条

### 错误状态

- 提示位置：评分卡内部显示「评分失败」
- 是否允许重试：是（刷新按钮或重新进入）
- 是否保留当前上下文：是

## 8. 设计注意点

- 视觉重点：AI 总分用大号数字 + 色彩等级（绿/黄/红）；废片标签用红色/橙色小标签
- 不要过度设计的地方：五维子分用简洁横条，不需要雷达图
- 必须清晰的地方：连拍组内 AI 推荐标记要醒目但不刺眼

## 9. 开发注意点

- 哪些可以先 mock：无（`aiScore` 与 `issues` 数据已有）
- 哪些交互先做最简版：「采纳 AI 推荐」先做单组操作，不做跨组批量
- 哪些动画先不要做：评分卡展开收起动画
