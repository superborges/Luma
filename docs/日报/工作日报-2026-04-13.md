## 今日已完成

### 全局交互与手感（P0 → P2）

- **P0**：`ContentView` 根布局统一 `StitchPressScaleButtonStyle`；主窗口 Sheet（导出面板、项目库、性能诊断）及 **设置窗口**（`LumaApp`）同样挂载，避免弹层与主窗手感不一致。
- **设计模块**：新增/扩展 `Sources/Luma/Design/StitchInteraction.swift`  
  - 主按钮 hover 略降不透明度（`stitchHoverDimming`）  
  - Recent 渐变图层的缩放 hover（`stitchImageHoverScale`）  
  - Stitch 语义下列表行背景 hover（`stitchListRowHoverBackground`）  
  - 网格卡浅色 overlay hover（`stitchSubtleCardHover`，支持圆角参数）  
  - 顶栏圆钮（`StitchToolbarIconCircleButton`）  
  - 系统材质上的中性行 hover（`stitchAdaptiveListRowHover`）  
  - Culling 侧栏未选中行 wash（`stitchUnselectedHoverWash`）  
  - 缩略图/连拍封面短缩放（`stitchAssetPreviewHoverScale`）

### Library / 主导航

- **SideNavBar**：导航项与底部设置齿轮的 hover 背景；结构保持与 Stitch 稿一致。
- **LibraryHubView**：顶栏搜索框 focus 描边；`New Expedition` / `View All` 等链接与 CTA 的 hover；Recent Bento 大图/小图缩放参数；All Expeditions 列表行、底部 Gallery 入口行、网格单元、布局切换 segment 的 hover；Vault 区 `Run Deep Integrity Scan` 与 archive 细分行的交互强化。
- **Recent 验证用资源**：打包 `Resources/recent-expedition-demo.jpg`，**仅 Recent 左侧大卡（featured）** 使用真实照片 + 原有 `stitchImageHoverScale`；渐变层 `allowsHitTesting(false)`，便于在半幅画面上触发底层图的 hover 缩放。

### 其他页面与面板

- **ImportsHubView**：管线阶段行、会话列表行的自适应行 hover；主操作 `borderedProminent` 组上逐钮 hover dimming。
- **ExportHubView**：最近导出列表行 hover；主 CTA hover dimming。
- **Culling**：`GroupSidebar` 未选中行 hover；`PhotoGrid` 缩略图/连拍/条带芯片的预览图短缩放；底部浮动工具条药丸与评星 Menu 的 hover dimming。
- **DetailPanel**：`detailRow` 统一行级 hover。
- **ProjectLibraryView**：All Expeditions Gallery 列表/网格与此前一致；管理模式下工具栏与详情区按钮的 hover dimming。
- **ExportPanelView**、**PerformanceDiagnosticsView**、**SettingsView**、**AIModelConfigView**：工具栏/表单按钮与统计行等按同一套修饰符补齐。

### 工程与验证

- `Package.swift` 为 Luma 目标增加 `Resources` 的 `.copy` 配置；`swift build` / `swift test` 在相关改动后通过。
- 多次本地 **重启 Luma**（`swift run`）便于即时查看效果。

## 待办（明天及后续）

### 高优先（UE 链路）

1. **All Expeditions 全流程**
  - 从 Library「View All / Gallery 入口 / 布局切换」进入 **全量远征列表或网格** 的路径：标题、空状态、溢出、返回/关闭后的状态是否与主导航一致。  
  - 与 **ProjectLibraryView**（管理态 vs Gallery 态）的 **命名、文案、工具栏、动效** 是否统一为同一套「远征」心智。
2. **Expedition detail（远征详情）**
  - 管理列表选中后的 **详情区**：信息层级、操作主次、危险操作确认、与 Culling/导入状态的联动说明。  
  - 若存在独立「详情页」或未来路由：过渡方式（sheet / push）、返回栈、深链是否需要预留。
3. **二级页整体 UE**
  - 加载中 / 错误 / 只读目录等 **反馈文案与可恢复动作**。  
  - **键盘与焦点**：Sheet 内 Tab 顺序、搜索框、列表选中与系统 `List` 行为是否满足专业用户预期。  
  - **空状态与引导**：无远征、无附加远征、导入中等场景的下一步操作是否清晰。

### 中优先（手感与实现债）

1. **Recent 整卡 `group-hover`**（若仍不满意）
  - 将 hover 状态提升到整张卡片 `Button`/`ZStack`，保证光标在标题区也能驱动底层图缩放（完全对齐 HTML 行为）。
2. **菜单栏命令（`LumaCommands`）**
  - 系统菜单项无法套用 SwiftUI `ButtonStyle`；若需一致「手感」，仅能通过文案/快捷键说明或自定义窗口内入口补强。
3. **项目库管理态左侧 `List`**
  - 系统 `List` 行 hover 与自定义行的差异；是否要为 `ProjectRow` 做与 Library 列表一致的自定义行样式。

### 低优先 / 备忘

1. **状态栏（`StatusBarView`）**
  - 当前以展示为主；若未来需要可点击筛选/跳转，再单独设计交互与动效。
2. **单张模式 `SingleAssetView`**
  - 未加预览图 hover 缩放，避免与缩放/滚动手势冲突；若产品希望有轻微反馈再评估。
