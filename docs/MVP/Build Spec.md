## 2. 本阶段解决的核心问题

- 从多种来源**导入**到本地项目，**智能分组 + 连拍折叠**后**选片**，再**导出**到文件夹 / Lightroom 目录 / Mac「照片」App，并可选处理未选照片与 Photos 源清理策略。

## 3. 已实现能力（菜单与主路径）

### 3.1 导入（Import Session 来源）

| 来源 | 实现状态 | 说明 |
|------|----------|------|
| 普通目录 | ✅ | `NSOpenPanel` 选择目录 |
| SD 卡 | ✅ | 依赖卷挂载检测；未检测到卡时菜单项禁用 |
| **Mac · 照片** | ✅ **（仅张数）** | 单页 `NSAlert` + 下拉选「最近 N 张」；**全库、按时间倒序、不选相册**；仅读**本地已缓存**资源，不主动拉 iCloud。 |
| iPhone · USB | ✅ | ImageCapture 路径；未连接/未信任时禁用 |
| 继续未完成导入 | ✅ | 存在可恢复 session 时显示 |

**实现备注（避免文档与 UI 不一致）**

- 工程内仍包含「完整版」**相册 + 时间 + 媒体类型 + 去重** 的 `AppKitPhotosImportPicker`（`NSAlert` 多段控件），**当前默认菜单未接入**；实机问题与**相册选择路径**相关，MVP 默认改为仅张数以降低风险。恢复前需单独评审与测试。

### 3.2 选片

- ✅ **Session**（原 PRD 称 Import Session）列表 → 进入 **选片工作区**（`CullingWorkspaceView` 三栏布局）。
- ✅ **智能分组**（时间/地点等）、**连拍**识别与组内选优流程。
- ✅ **Picked / Rejected / Pending** 与键盘快捷键。
- ✅ **本地**启发式/规则评分（`LocalMLScorer` 等），右栏可展示分数类信息；**无云端 Vision API 路线**（产品不规划实现）。

### 3.3 导出

- ✅ 目的地三选一：**文件夹**、**Lightroom（自动导入目录 + XMP）**、**照片 App**。
- ✅ 未选照片处理：**缩小保留** / **归档视频** / **忽略**（与 `RejectedHandling` 一致）。
- ✅ 源含「照片库」时：**清理策略**（保留原图 / 按策略回删等）、系统确认对话框链路；**试跑（dry-run）** 等开关以代码为准。
- ✅ 导出任务与进度/摘要（`ExportJob`、`ExportSummaryView` 等）。

### 3.4 其他界面

- ✅ 首页 = **项目 / Session 列表**（`SessionListView`）：排序、新建导入、行内打开/归档/菜单。
- ✅ **项目库**、**新用户引导**、**系统设置**（通用 / 导出默认值 / 开发诊断）。
- ✅ 图库只读、有限访问等：SwiftUI `alert` 说明 + 跳转系统隐私；详见 `PhotosAccessGuidance`。
- ✅ 诊断：`PerformanceDiagnosticsView`、运行时 trace、`import-breadcrumb.jsonl`（导入阶段同步落盘）。

### 3.5 明确未纳入主目标 / 非 MVP

- 云端多模型 AI 评分与按预算调度（`docs/raw/PRODUCT_SPEC.md` 中部分章节为**愿景**，不开发）。
- 设置页中 **无**「多模型 / API Key / 经济·平衡·最高质量」等完整配置（长期愿景见 `docs/raw/PRODUCT_SPEC.md` 与 `docs/MVP/Concept Note`）。

## 4. 用户主路径（与实现一致）

1. 打开 App → **Session 列表**。
2. **新建** → 选择导入来源（目录 / SD 卡 / **Mac 照片·仅张数** / iPhone）→ 导入完成进入选片。
3. 在**选片页**完成决策 → **导出** → 选目标与选项 → 执行。
4. 可回到列表继续其他 Session 或**归档**（以 UI 行为为准）。
