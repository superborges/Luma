# UI flow Note - 导出页

## 1. 流程名称

- 名称：导出
- 所属模块：导出页
- 使用频率：中

## 2. 这个流程要解决什么

- 用户来到这个流程时的目标：把一个 Session 里选中的照片导出到目标位置；同时给未选中的照片一个体面的归宿（归档成视频做纪念）；如果源是 Photos 库还要顺手清爽手机相册
- 用户最关心的信息：导出去哪里、导出格式、未选中照片怎么处理、是否会动到原 Photos 库
- 用户最害怕发生的问题：
    - 导出了一半失败，再点要从头开始
    - 误导出到错误目录覆盖了原文件
    - 未选中的照片直接删掉，事后后悔
    - 导出回 Photos 把原图全删了找不回来

## 3. 入口与出口

- 从哪里进入：选片页右上角"去导出"按钮 / 首页 Session 列表的"导出"快捷入口
- 完成后去哪里：首页（Session 状态变为"已导出"）
- 中途取消后回到哪里：选片页（导出配置自动保存）

## 4. 页面步骤

### Step 1

- 页面名：导出页 - 配置目标
- 用户要做什么：选择导出目的地，配置输出格式
- 系统展示什么：
    - 导出目标三选一：
        - **普通目录**：选个文件夹，导出 JPEG/HEIC/RAW
        - **Lightroom 自动导入目录**：兼容 LR 工作流
        - **Mac·照片 App**：写入照片库，iCloud 自动同步到 iPhone
    - 选中数量统计：X 张选中将被导出
    - 输出格式：JPEG / HEIC / 原始 RAW
    - 文件命名规则：保留原名 / 按日期重命名 / 自定义模板
    - **当目标是 Mac·照片 App 时，额外显示**：
        - 相册命名（默认 = Session 名）
        - 是否按分组创建子相册
- 主按钮是什么：选择目录的"浏览"按钮（普通目录/Lightroom 时）

### Step 2

- 页面名：导出页 - 未选中照片处理
- 用户要做什么：决定 Reject 和 Pending 照片的去向
- 系统展示什么：未选中数量统计（Y 张），三种处理方式：
    - 丢弃：保留在 Session 里不动
    - 归档为视频：按分组把未选中照片压缩成短视频，留作纪念
    - 缩小保留：转成低分辨率 JPEG 归档，省空间但可看
- 主按钮是什么：单选三种方式之一

### Step 3

- 页面名：导出页 - 清理源相册（仅当源 = Mac·照片 App / iPhone·USB 时出现）
- 用户要做什么：决定是否清爽原 Photos 库
- 系统展示什么：两种策略 + 预览：
    - 仅加新相册：在 Photos 库里新建相册装 Picked，**原图全部保留**（最安全）
    - 加相册 + 删除未选原图：导出 Picked 到新相册后，对 Reject 的原图发起系统删除请求，**会弹系统对话框由你最终确认**
    - 预览：将创建相册 X 张 / 将申请删除 Y 张
- 主按钮是什么：单选两种策略之一

### Step 4

- 页面名：导出页 - 执行
- 用户要做什么：确认配置，触发导出
- 系统展示什么：
    - 导出预览摘要：X 张到 [目标]，Y 张 [处理方式]，Z 张 [清理策略]
    - 进度条：当前进度 / 总数
    - 实时反馈：当前正在导出的文件名
    - **当源 = Mac·照片 App 且当前需要原图时**：进度条上额外显示"正在从 iCloud 下载原图 N/M"（首次需要原图的时机才下载，避免选片阶段就拉云端）
- 主按钮是什么："开始导出"

### Step 5

- 页面名：导出页 - 完成
- 用户要做什么：确认导出结果
- 系统展示什么：
    - 成功摘要：导出 X 张到 [目标]；归档 Y 张到 [视频路径]；清理原图 Z 张
    - 失败列表（如果有）：哪些文件失败、原因
- 主按钮是什么：
    - "打开目标目录" / "打开 Photos 库"
    - "返回首页"

## 5. 数据规则

- 一个 Session 可以多次导出（每次导出生成一条 ExportJob 记录）
- 导出配置（目标/格式/未选中处理/清理策略）默认从用户上次配置继承
- 导出失败的文件保留在 Session 里，下次导出可只重试失败项

## 6. 与实现的对应关系（`ExportPanelView`）

- 导出目标 Picker：**文件夹** / **Lightroom** / **照片 App**（与文内 Step 1 一致）。
- **文件夹**目标：目录模板、XMP、修图建议、输出目录等；**未选照片处理** 在单独 Section，文案为 缩小保留 / 归档视频 / 忽略（与 `RejectedHandling` 中文标签一致）。
- **照片 App** 目标：按分组建相册、合并 RAW+JPEG、保留 Live、AI 说明字段等；其中「写入 AI 评语到说明」在 UI 中有 **macOS/PhotoKit 限制** 的提示，以界面为准。
- 若同时存在 `onlyAssetIDs`：界面会提示**仅重试失败项**（见代码）。
- **首页**「去导出」与选片内入口以实际 `ContentView` / 工具栏为准。

*Step 中未单独拆页：配置与执行为一连续表单 + 任务，不强制多 Step 页面。*  
- 删除原图通过 PhotoKit `PHAssetChangeRequest.deleteAssets` 的，**Apple 强制系统确认**，Luma 不绕过。

### 本阶段不做的

- 导出到云盘（iCloud Drive / Dropbox 等）
- 导出后自动通知（邮件/系统通知）
- 自定义水印
- 跨多个照片图库写入
