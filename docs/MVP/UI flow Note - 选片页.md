# UI flow Note - 选片页

## 1. 流程名称

- 名称：选片
- 所属模块：选片页
- 使用频率：高

## 2. 这个流程要解决什么

- 用户来到这个流程时的目标：在一个 Import Session 里把好照片挑出来，把不要的拒掉，剩下的留待定
- 用户最关心的信息：每张照片的清晰度/曝光/构图、连拍组里哪张最好、AI 推荐的选中、当前选片进度
- 用户最害怕发生的问题：
    - 几百张照片一张张点太累
    - 连拍 5 张分不清差异，反复横跳
    - 选完没保存，重启就没了
    - AI 推荐和自己感觉差太多，反而干扰判断

## 3. 入口与出口

- 从哪里进入：首页点击一个 Import Session
- 完成后去哪里：导出页
- 中途取消后回到哪里：首页（选片状态自动保存）

## 4. 页面步骤

### Step 1

- 页面名：选片页 - 进入
- 用户要做什么：浏览 AI 智能分组，选择一个分组开始选片
- 系统展示什么：
    - 左栏：智能分组列表（AI 按时间/地点聚类的组），每组显示缩略图、组名、照片数、选片进度
    - 中间区：默认显示第一个分组的照片网格
    - 右栏：当前选中照片的元信息（EXIF/AI 评分/修图建议）
- 主按钮是什么：点击左栏分组进入该组

### Step 2

- 页面名：选片页 - 单张操作
- 用户要做什么：在分组内对照片做选中（Pick）/ 拒绝（Reject）/ 待定（Pending）操作；可切换网格/单张模式
- 系统展示什么：
    - 中间区：
        - 网格模式：照片缩略图，连拍组堆叠展示，AI 推荐照片有标记
        - 单张模式：大图预览，可上下张切换
    - 右栏：当前照片的 AI 评分（清晰度/构图/曝光等子项）+ 修图建议文本 + EXIF
    - 底栏：当前组进度条（已选 X / 拒 Y / 待定 Z）
- 主按钮是什么：
    - 选中（快捷键 P）
    - 拒绝（快捷键 X）
    - 待定（快捷键空格）

### Step 3

- 页面名：选片页 - 连拍组内选优
- 用户要做什么：在一个连拍组里挑出最佳一张
- 系统展示什么：连拍组内所有照片并排展示，AI 推荐的"最佳"那张高亮
- 主按钮是什么："采纳 AI 推荐"，或手动点选某张为最佳

### Step 4

- 页面名：选片页 - 完成
- 用户要做什么：完成所有分组选片，进入导出
- 系统展示什么：顶部显示总进度（X 张已决策 / 共 Y 张），全部完成后右上角"去导出"按钮高亮
- 主按钮是什么："去导出"

## 5. 关键交互/快捷键

- P：选中
- X：拒绝
- 空格：待定
- ←/→：上一张/下一张
- ↑/↓：上一组/下一组
- G：切换网格/单张模式
- 1-5：评分 1-5 星（可选）

## 6. AI 能力在本流程的体现（与实现）

- **智能分组**：进入选片页前由分组引擎完成（时间/地点等策略以代码为准）。
- **连拍识别**：连拍子组与组内选优，与 `GroupingEngine` / UI 表现一致。
- **打分**：**本地** `LocalMLScorer` 等路径为主；**无** 设置页中的云端多模型、预算、经济/平衡策略（非 MVP 主工程）。
- **修图建议 / 长文本**：以右栏/资产字段实际展示为准；云端 Vision 增强非主路径。
