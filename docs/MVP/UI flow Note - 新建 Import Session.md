# UI flow Note - 新建 Import Session

## 1. 流程名称

- 名称：新建 Import Session
- 所属模块： 首页
- 使用频率：中

## 2. 这个流程要解决什么

- 用户来到这个流程时的目标：新建一个 Import Session
- 用户最关心的信息：来源目录/空间消耗/支持的类型
- 用户最害怕发生的问题：
    - 在大容量目录时性能和稳定性
    - 从 iPhone/iCloud 导入时把云端原图全拉下来，磁盘炸了
    - 不知道当前导入的是 iCloud 同步过来的版本还是原图

## 3. 入口与出口

- 从哪里进入：首页
- 完成后去哪里：选片页
- 中途取消后回到哪里：首页

## 4. 页面步骤

### Step 1

- 页面名：首页
- 用户要做什么：创建一个新的 Import Session
- 系统展示什么：跳转到新建页面，包括来源目录：
    - 普通目录
    - SD 卡（自动检测插入的卡）
    - **Mac·照片 App**（默认的"从 iPhone 导入"路径，含 iCloud 同步过来的所有照片）
    - iPhone·USB 直连（高级选项，仅在检测到设备时高亮）
- 主按钮是什么：「新建 Import Session」按钮（弹出菜单，选来源）

### Step 2

- 根据来源不同：
    - **普通目录 / SD 卡 / iPhone USB**：系统文件选择或设备流程 → **导入进度**（与实现一致）。
    - **Mac · 照片（当前实现）**：**不展示**独立「新建页」；授权通过后弹出 **单页系统对话框**，仅选择 **「最近 N 张」**（N 为预设档位），**无** 相册/时间/媒体类型等筛选。导入阶段展示进度。  
- *（产品愿景中曾规划「时间范围筛选、相册」等，见 `docs/raw/PRODUCT_SPEC.md`；主工程内另有未默认启用的完整 `NSAlert` 实现。）*

### 导入中

- 系统展示什么：源类型、进度；完成后进入选片工作区

## 5. 来源差异说明

| 来源 | 适用场景 | 关键约束 |
|---|---|---|
| 普通目录 | 已经整理过的本地照片库 | 原图必须在本地 |
| SD 卡 | 相机拍完直接读卡 | 插入即检测，断开提示 |
| Mac·照片 App | iCloud 同步过来的 iPhone/iPad 照片 | **只读本地缓存版**；MVP 菜单路径为 **仅选张数、全时间倒序**；不主动在导入阶段从 iCloud 拉原图，导出需原图时再拉（与实现一致） |
| iPhone·USB 直连 | iCloud 没开 / 想要原图 / 一次性导入新机 | 走 ImageCaptureCore，需手机解锁并信任 |

## 6. mvp 不做的

- 网盘 / NAS 直接导入
- 后台增量监听 Photos 库（每次手动新建 Session）
