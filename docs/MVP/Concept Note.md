# Concept Note

## 1. 产品名称

- 名称：Luma  拾光

## 2. 一句话定义

- 这是一个帮助 [摄影爱好者] 在 [旅行或者活动结束之后] 下更快完成 [相机、手机照片导入、挑选、统一风格 P 图、按照规范目录整理大量照片的] 的产品。

## 3. 目标用户

- 核心用户：我自己（一个爱好摄影的软件设计师，每年有 4-5 次旅行，使用 iPhone、富士 X-T5、徕卡Q 相机），有两个娃，希望给娃和他们的妈妈拍美照，喜欢拍风景。
- 次级用户：我老婆（一个设计师），爱美，喜欢自拍，喜欢拍美食，喜欢拍娃。
- 为什么他们会愿意用：拍照很开心，但选片很痛苦，修片也很痛苦。“拍照一时爽，修片火葬场”

## 4. 核心问题

- 用户当前是怎么做这件事的：使用 lightroom。或者在手机相册里使用收藏功能。
- 现有方式哪里痛苦：lightroom 收费，且有点重，和 iPhone 生态不打通。
- 这个痛点为什么值得解决：解决了之后我出去旅行的时候就可以尽情的拍照了，拍娃和娃妈，拍风景，拍风土人情。

## 5. 核心价值

- 我提供的关键能力是：在常规导入导出的基础上增加对iPhone 相册导入导出的支持，方便把手机相册整理成精美的画廊。在常规选片基础上增加连拍组选片/智能打分的支持。
- 对用户最直接的收益是：照片整理更方便/更智能；可能把手机相册整理的很整洁，风格统一。

## 6. 为什么现在做

- 为什么这个产品在现在值得做：我一直在找一款符合我需求的软件，但始终没有找到最满意的，现在 AI Coding 能力这么强，我又有一些开发的能力，所以觉得可以做了。
- AI/工具/个人能力给了什么新机会：在产品功能上，可以智能分组，智能选片，智能提供照片修改建议；甚至可以智能修图（我现在不确定模型能力是否能做到，需要更多的调研）

## 7. 与当前实现的关系（MVP 基线）

- 上文「智能分组、连拍、AI 建议」在 **用户价值** 上仍成立；**主工程**以 **本机选片 + 本地打分辅助 + 多目标导出** 为主，**不开发** 云端多模型与预算类能力。文档与实现以 `docs/MVP/Build Spec.md` 为对齐锚点；`docs/raw/PRODUCT_SPEC.md` 中涉及云端评分的章节视为长期愿景/历史规格。
