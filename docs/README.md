# Luma 文档索引

| 目录 / 文件 | 用途 |
|-------------|------|
| **`raw/`** | **原始需求与长期愿景**：完整产品规格、不受当前实现裁剪。以 `PRODUCT_SPEC.md` 为主。 |
| **`MVP/`** | **与当前代码一致（可维护）**：`Build Spec.md` 为 **MVP 实现基线**；各 `UI flow Note` 描述界面与流程，**与实现冲突时以代码为准，并应回头改文档**。 |
| **`日报/`** | 工作日报存档，不保证与当前行为一致。 |

**维护约定**

- 产品阶段从 **MVP 可运行闭环** 进入 **v1 功能迭代** 时，优先更新 `MVP/Build Spec.md` 和受影响的 `UI flow Note`。
- 代码中有意保留但未接入菜单的路径（例如「完整版」照片 `NSAlert` 选相册）在 `Build Spec.md` 的「实现备注」中标注，避免和菜单文案混淆。

**最近对齐（文档批次）**

- 见 `MVP/Build Spec.md` 文内「修订记录」。
